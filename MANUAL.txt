🔧 1. Hardware Overview
RP2040 Pico
Runs main.py

Generates sweep data

Handles TUNE, SWEEP, REPLAY

Outputs binary RF samples over USB serial

CC1101
RF front‑end

Tuned by Pico via SPI

Provides RSSI / raw sample data

Linux Host (Kali)
Runs bridge.py

Exposes Pico data as a TCP device for SDR++

Auto‑detects /dev/pico or /dev/ttyACM*

SDR++
Displays waterfall

Reads from TCP input

Shows live RF spectrum

🔌 2. Connecting the Pico
Check device
bash
ls -l /dev/pico
Should show:

Code
/dev/pico -> ttyACM1
Open REPL
bash
picocom /dev/pico -b115200 --echo --imap crlf
Exit:

Code
Ctrl+A  Ctrl+X
🧠 3. Pico Firmware Commands (main.py)
When connected via picocom, type these commands:

Available Commands
Code
SWEEP
TUNE <freq>
REPLAY
BIN_ON
BIN_OFF
Descriptions
SWEEP
Runs a frequency sweep across the configured band.
Used for waterfall scanning.

TUNE <freq>
Example:

Code
TUNE 433920000
Locks CC1101 to a single frequency.

REPLAY
Replays the last captured RF burst (if implemented in your build).

BIN_ON
Enables binary mode (raw sample streaming).

BIN_OFF
Disables binary mode (text mode only).

⚡ 4. SPEEDING UP THE WATERFALL (IMPORTANT)
Edit sweep delay in main.py

Code
time.sleep(0.05)

Code
time.sleep(0.001)

Code
time.sleep(0)

🛠️ 8. Troubleshooting
Bridge says “Address already in use”
Kill old instance:

bash
sudo fuser -k 9100/tcp
No data in SDR++
Check:

bash
picocom /dev/pico
Send:

Code
BIN_ON
SWEEP
Pico not detected
bash
ls -l /dev/pico
If missing:

bash
sudo udevadm control --reload-rules
sudo udevadm trigger
